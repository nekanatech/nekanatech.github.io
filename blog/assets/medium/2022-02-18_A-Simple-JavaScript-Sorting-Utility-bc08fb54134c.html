An abstraction of the solution

I pondered a few strategies and eventually decided to use something of a majority vote strategy that was initially devised by Boyer & Moore (1991). The original algorithm iterates through an array or list and counts the frequency of the different kinds of items in the list. A “winner” is declared and outputted onto the screen.

I made a sorting algorithm that uses a similar kind of reasoning, where I declare a “winner” by iterating through all numbers in a list, the “input list,” and setting the winner to the smallest number. This “winner” then gets removed from the input list and then appended onto another list, the “output list.” This process repeats until the input list is empty and then the output list is returned to the function that called it.

    // sorting logic goes here
};

The first line (1) assumes that the first number in the input list is the smallest. A for loop is setup (2) where every number in the input list is iterated through and compared to the winner (3). If the number in the input list is indeed smaller than the winner, then the winner is set to that number (4). This process is repeated until the entire list is iterated through. The winner is then pushed onto the output list (5) and removed from the input list (6).

However, this for loop will only remove one number from the input list and append it to the output list. We need to do this for all numbers in the input list. To accomplish this, a do…while loop is employed as the second part of the main logic:

Showcasing the sorting implementation

I created bad_sort in part to practice thinking like a programmer. Engineering is all about finding solutions to problems, and software engineering is no exception. Furthermore, finding solutions to problems takes creativity and cleverness, with software engineering, once again, not being an exception.

Of course, programmers and software engineers should not “reinvent the wheel” (Althoff, 2016) too much and should use other libraries when working on professional programming jobs (Ibid). This is just a case study for thinking of a solution to a problem that may be novel for someone learning how to code for the first time.
