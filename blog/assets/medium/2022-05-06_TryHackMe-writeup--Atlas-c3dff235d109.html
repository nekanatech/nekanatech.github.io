Procedure

Before I start, I edited the /etc/hosts file on my AttackBox and added the following line:

<target ip> atlas.thm

I did this because I know that I will not finish the room in one sitting and that the IP address of a TryHackMe boot2root machine will change for each new session. It would be better to record a pseudo-domain name as opposed to a dynamic IP address, so that every time I boot up the target VM, I can just edit the /etc/hosts file with the new target IP address.

I then clicked on the green-coloured ‚Äústart machine‚Äù button that is on the top-right corner of the first task and proceeded to probe the machine.

The room notes that the target machine is running Windows, so the -Pn flag will need to be used to ignore the fact that Windows does not respond to ICMP requests and proceed to launch a port scan regardless. The -oX tcp_scan.1.xml flag instructs nmap to store its results in a XML format.

Fig. 1 shows an excerpt of the xsltproc output‚Ää‚Äî‚Ääspecifically the open ports on the target system:

There are two ports open: one TCP-driven service on 3389 and another TCP-driven service on port 8080. The scan also identified the operating system as ‚ÄúAVtech Room Alert 26W environmental monitor‚Äù.

Note that the service on 7680 has been tcpwrappped, which means that whatever is running it is referencing the /etc/hosts.allow and /etc/hosts.deny files and grants (or denies) access based on their configurations (Red Hat Customer Portal, n.d). It should also be noted that a tcpwrapped service is different from firewall ‚Äîsomething that is discussed in Attrition.org (c.a. 1998).

Further inspection of the nmap results shows what the services on port 3389 and 8080 are running respectively. The former is running the remote desktop protocol service (as indicated by the Microsoft Terminal Services banner; also see Liang et al., 2021), whilst the latter is running the ThinVNC service (Cybele et al., n.d.)‚Ää‚Äî‚Ääas noted by the following snippet from the nmap results: [‚Ä¶] ticate: Digest realm="ThinVNC", qop="aut [‚Ä¶].

Summary

Despite this being an easy ‚Äútutorial room,‚Äù it took longer than it was supposed to because of my somewhat odd habit of deviating a bit from the instructions given by the room‚Äôs author. Hacking is all about breaking the rules and finding novel and clever solutions to conventional problems‚Ää‚Äî‚Ääespecially in an offensive security problem.

Nonetheless, I have gone about using the PrintNightmare exploit as the author intended in Appendix A.2 for anyone interested. I have also manually exploited the ThinVNC vulnerability vector in Appendix A.1 for any overachievers.

Takeaways

  • Proof-of-concepts can sometimes be broken, so setup a virtual machine lab and make sure to test them first.
  • If proof-of-concept is broken, try to debug it, find a more reliable implementation online, or even engineer it yourself. I went with the second and (sort of) third for exploiting ThinVNC.
  • Unless you are taking the Offensive Security Certified Professional exam, don‚Äôt be afraid to use Metasploit and Meterpreter to make your life easier¬†;-)

A.1 Manual exploitation

‚ÄúMurilandOracle‚Äù (2022a, task 4) challenged the contender to execute the ThinVNC exploit manually. To get an idea of how the exploit that he engineered works, I will examine the relevant bits of its source code. The following is an excerpt or MurilandOracle‚Äôs fixing of the broken proof-of-concept:

The important bits of MurilandOracle‚Äôs exploit is that it takes in arguments from the command line and stores the target URL is stored as a string in url. The port to attack is stored as an integer in the port variable.

The url uses an http:// or https:// depending on whether-or-not the ssl argument is used. The exploit then makes a HTTP request with a GET prepared statement. Specifically, the request that the exploit will make is:

http://atlas.thm:8080/abc/../../ThinVNC.ini

I ‚Äúreproduced‚Äù the exploit manually by opening up a python3 shell and manually typing in commands after MurilandOracle‚Äôs exploit:

>>> url_to_attack = "http://atlas.thm:8080/abc/../../ThinVNC.ini"
>>> request = requests.Request(method="GET", url=url_to_attack)
